<analysis>**original_problem_statement:**
The user wants to build a full-stack pharmacy delivery service application named RX Expresss, which should replicate the functionalities of the DrugLift app. The only exception is that all routing, delivery management, and proof-of-delivery functionalities will be handled by integrating a third-party service, Circuit Spoke.

**PRODUCT REQUIREMENTS:**
- **Core Goal:** Replicate DrugLift's feature set.
- **Key Features:**
  - Multi-role authentication (Admin, Pharmacy, Driver, Patient).
  - Pharmacy Portal: Dashboard, creating/managing deliveries (with pricing visibility, notes, and QR code generation), QR code scanning.
  - Driver Portal: View and manage assigned deliveries, scan packages, capture Proof of Delivery, mark copay as collected.
  - Admin Dashboard: An advanced, interactive dashboard for full operational control. This includes a Smart Organizer for grouping orders, drag-and-drop route assignment, route optimization, real-time driver tracking, user management with editing, and quick-print labels.
  - Order Status Management: A simplified, meaningful, and easy-to-use status update system.
  - Visual Indicators: Icons for special package types like refrigerated items.
- **Mandatory Integrations:** Circuit Spoke, Google Maps, Stripe, Twilio, SendGrid.
- **QR Code System:** Borough-prefixed QR codes generated for each order.
- **Proof of Delivery (POD):** Drivers must be able to capture photo and digital signatures.

**User's preferred language**: English
